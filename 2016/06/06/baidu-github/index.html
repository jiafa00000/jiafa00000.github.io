解决github pages无法被百度爬虫抓取

Posted by Kazila on 2016-06-06

背景

github屏蔽了百度爬虫，因此百度不能自动收录github托管的hexo博客。
在知乎上大体找到两种方案

方案一

用CDN加速，由CDN来代理github pages的博客。有局限性，节点的距离不同而造成403.
原文如下：

作者：邹润阳
链接：https://www.zhihu.com/question/30898326/answer/50084979
来源：知乎

使用 CDN 来代理 Github Pages 上的博客依旧不能很好地解决百度爬虫爬取不到的问题。
因为使用 CDN 的原理是让 BaiduSpider 访问 CDN 的代理服务器，而不直接向 Github 发送请求。
现在我们做这样一个假设，你所使用的 CDN 为你的网站分配了 6 个节点，A - B - C - D - E - F
首先你访问你的网站，会请求离你最近的节点 A。 节点 A 发现自己没有你要的资源，就用你的 User-Agent 向 Github 发送请求，这样获取到了内容，节点 A 就有你的博客的缓存了。节点 A 附近的请求都可以受益于这次缓存。
但是如果 BaiduSpider 所处的位置离 B 节点比较近，而 B 节点没有缓存。此时 BaiduSpider 发出请求时，节点 B 就会用 BaiduSpider 的 User-Agent 去向 Github 请求内容。然后就 403 Forbidden了。。
所以利用 CDN 来使得百度爬虫能够爬取 Github Pages的方法有一定局限性。当然，如果你的博客访问量很大，比如节点 B 有正常用户访问过你的博客，那 BaiduSpider 也就能够获取到正常的内容了。因为不需要直接与 Github 交互。

方案二

同步到gitcafe。gitcafe最近已被coding收购，直接访问
大体思路就是让push到github上的文件自动同步到gitcafe上，然后可以考虑更改域名解析设置，让国外解析到github上，国内解析到cafe上。
先注册coding，建项目名字和和用户名一样，初始化项目，创建分支，增加ssh。
dnspod增加两个：
@ CNAME 国内 username.coding.me.
www CNAME 国内 username.coding.me.
不说了我去捣鼓了。。。
